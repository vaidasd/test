• Frames can have a name and are constructed with one if required
    • A Frame without a name is anonymous, and can be useful for testing
    • You can ask Frames for their name
    • Frame itself is generally abstract but we don’t care about this constraint. Sometimes it can be useful to create one
    • We indicate later in this document, using in a non-Smalltalk syntax, making certain types of Frames with given names. You need to translate this to your specific implementation.
